## 2. 技术选型说明

- **前端框架**：React 18 + TypeScript（类型安全，组件化开发
- **构建工具**：Vite 5（快速构建，热更新
- **路由管理**：react-router-dom 6（SPA路由）
- **状态管理**：zustand 4（轻量级状态管理
- **样式方案**：TailwindCSS 3（原子化CSS
- **图标库**：lucide-react（线性图标库
- **图表库**：recharts（React图表组件
- **二维码**：qrcode.react（二维码生成
- **数据持久化**：localStorage（本地数据存储
- **Mock数据**：内置TypeScript类型定义驱动
- **后端**：无（纯前端H5，数据存本地存储

## 5. 核心算法设计

### 5.1 采血位择优分配算法（避免碎片+负载均衡）

```
算法：智能分配采血位
输入：日期date，时间段timeSlot，预约列表appointments，采血位列表stations
输出：最优采血位stationId

1. 获取该时间段所有采血位的预约数
2. 过滤出空闲容量 > 0 的采血位
3. 计算每个采血位的分配评分：
   score = (剩余容量权重) * 0.4 + (负载均衡权重) * 0.3 + (碎片避免权重) * 0.3
   
   - 剩余容量：优先选择剩余容量较多的
   - 负载均衡：优先选择今日预约数较少的（避免集中在某几个采血位
   - 碎片避免：优先选择与该时间段前后已有预约的（合并预约）

4. 返回评分最高的采血位ID
```

### 5.2 献血间隔校验算法

```
规则：
- 全血献血间隔：≥ 180天（6个月
- 成分血献血间隔：≥ 14天
- 全血后成分血：≥ 90天
- 成分血后全血：≥ 28天
```

## 7. 关键实现要点

### 7.1 移动端适配方案
- 使用 `rem` + `viewport` + `TailwindCSS`响应式类
- 基准宽度：375px设计稿
- 关键断点：sm(640px), md(768px)

### 7.2 性能优化
- 组件懒加载（React.lazy
- 列表虚拟滚动（长列表
- Zustand选择性订阅（避免不必要重渲染
- 防抖节流处理（搜索、日历滚动

### 7.3 数据持久化
- Zustand `persist`中间件 + localStorage
- 数据自动保存和恢复

### 7.4 样式规范
- 医疗主题色配置在 `tailwind.config.js`中定义
- 使用 CSS Variables管理色系
- 统一间距、圆角、阴影等设计Token
